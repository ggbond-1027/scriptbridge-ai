# 贡献指南

## PR 提交规范

1. 请基于 PR 添加新功能或修复问题，避免直接在主分支堆叠未说明的变更。
2. 每个 PR 只做一件事：每个 PR 只实现或修改单一功能。大功能应拆分为多个独立 PR 分步提交。
3. PR 标题与描述需清晰完整，内容包含：
   - 标题：一句话说明本 PR 新增或修改了什么。
   - 功能描述：说明该功能的作用与使用方式。
   - 实现思路：简要说明技术选型或核心实现逻辑。
   - 测试方式：说明如何验证该功能正常运行。
4. PR 合并后，主分支代码需保持可运行状态，评委在任意时间查看应能复现演示效果。

## 推荐流程

1. 从 `main` 创建功能分支。
2. 完成单一功能或修复。
3. 运行相关测试或构建命令。
4. 按 `.github/PULL_REQUEST_TEMPLATE.md` 填写 PR 描述。
5. 合并前确认没有提交密钥、token、密码、私钥或本地运行数据。

## 常用验证命令

```powershell
python -m pytest apps/api/tests
npm --prefix apps/web run build
npm run test
```
